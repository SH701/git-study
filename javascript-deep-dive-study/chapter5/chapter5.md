# 5장 표현식과 문

## 5.1 값

- **값(value)**: 식(표현식)이 평가되어 생성된 결과
- 모든 값은 데이터 타입을 가지며, 메모리에 2진수(비트)로 저장됨

```javascript
10 + 20; // 30 (식이 평가되어 값 생성)
```

## 5.2 리터럴

- **리터럴(literal)**: 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법
- JS 엔진은 런타임(코드 실행 시점)에 리터럴을 평가해 값을 생성

| 리터럴 종류 | 예시 |
|------------|------|
| 정수 리터럴 | `100` |
| 문자열 리터럴 | `'Hello'` |
| 불리언 리터럴 | `true`, `false` |
| null 리터럴 | `null` |
| undefined 리터럴 | `undefined` |
| 객체 리터럴 | `{ name: 'Lee' }` |
| 배열 리터럴 | `[1, 2, 3]` |
| 함수 리터럴 | `function() {}` |

## 5.3 표현식

- **표현식(expression)**: 값으로 평가될 수 있는 문
- 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조

```javascript
var score = 50 + 50;  // 50 + 50은 평가되어 100이 되므로 표현식
score;                // 변수 식별자 참조도 값으로 평가되므로 표현식
```

> **핵심**: 값으로 평가될 수 있는 문은 모두 표현식이다.

## 5.4 문

- **문(statement)**: 프로그램을 구성하는 기본 단위이자 최소 실행 단위
- 문의 집합 = 프로그램
- 문을 작성하고 순서에 맞게 나열하는 것 = 프로그래밍

### 토큰(Token)

- 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소
- 예: 키워드, 식별자, 연산자, 리터럴, 세미콜론 등

### 문의 종류

| 종류 | 예시 |
|------|------|
| 선언문 | `var x;` |
| 할당문 | `x = 5;` |
| 조건문 | `if (x > 1) { }` |
| 반복문 | `for (;;) { }` |

## 5.5 세미콜론과 세미콜론 자동 삽입 기능

- 세미콜론(`;`)은 문의 종료를 나타냄
- 세미콜론은 생략 가능 (ASI: Automatic Semicolon Insertion)
- 단, **세미콜론 사용이 권장됨** (예상치 못한 동작 방지)

## 5.6 표현식인 문과 표현식이 아닌 문

### 구분 방법

**변수에 할당해 보기** - 할당이 가능하면 표현식인 문

```javascript
// 표현식이 아닌 문 (변수 선언문)
var x;
var foo = var x;  // SyntaxError: Unexpected token var

// 표현식인 문 (할당문)
x = 100;
var foo = x = 100;  // 정상 동작 (x = 100이 값 100으로 평가됨)
```

| 구분 | 예시 | 값으로 평가 |
|------|------|-------------|
| 표현식인 문 | `x = 100` | O (100) |
| 표현식이 아닌 문 | `var x` | X |
