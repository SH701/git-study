# 4장 변수

## 4.1 변수란?

- 메모리에 저장되는 모든 값은 2진수로 저장된다
- 변수는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 공간을 식별하기 위해 붙인 이름
- 변수에 값을 저장하는 것을 **할당(assignment)**, 읽어들이는 것을 **참조(reference)**라고 한다

## 4.2 식별자

- 변수, 클래스, 함수 등의 모든 이름을 **식별자(identifier)**라고 한다
- 식별자는 고유해야 한다
- 변수는 값이 아니라 **메모리 주소**를 기억하고 있다

## 4.3 변수 선언

자바스크립트 엔진은 변수 선언을 **2단계**로 수행한다:

1. **선언 단계**: 변수 이름을 메모리에 등록
2. **초기화 단계**: 메모리 공간을 확보하고 암묵적으로 `undefined`를 할당해서 초기화

- `var`는 선언과 초기화를 동시에 진행한다 (사용이 권장되지 않음)
- 선언되지 않은 식별자에 접근하면 `ReferenceError`(참조 에러)가 발생한다

## 4.4 변수 선언의 실행 시점과 호이스팅

```javascript
console.log(score); // undefined
var score; // 변수 선언문
```

- 모든 선언문을 소스코드에서 찾아내 먼저 실행하고, 선언문을 제외한 소스코드를 한 줄씩 순차적으로 실행한다
- 모든 선언문이 코드의 어디에 있든 다른 코드보다 먼저 실행된다
- 선언문이 코드의 최상단으로 끌어 올려진 것처럼 동작하는 JS 고유의 특징을 **변수 호이스팅(variable hoisting)**이라고 한다

## 4.5 값의 할당

```javascript
var score;
score = 80;

// 위 코드와 아래 코드의 값의 할당 시점은 같다
var score = 80;
```

- 변수 선언은 런타임 이전에 실행되지만, 값의 할당은 런타임에 실행된다

```javascript
console.log(score); // undefined
score = 80;
var score;
console.log(score); // 80
```

**이유**: 변수 선언이 호이스팅으로 가장 먼저 되고, 첫 번째 `console.log`는 값이 할당되기 전이므로 `undefined`, 두 번째 `console.log`는 80이 할당된 후이므로 80 출력

## 4.6 값의 재할당

```javascript
var score = 80;
score = 90;
```

- 변수의 값은 재할당될 수 있다
- 변수의 값을 변경할 수 없다면 변수가 아니라 **상수(constant)**이다
- 상수는 한번 정해지면 변하지 않는 값이다 (`const`)
- **가비지 컬렉터(garbage collector)**: 더 이상 사용되지 않는 메모리를 해제하는 기능으로, 메모리 누수를 방지한다

## 4.7 식별자 네이밍 규칙

1. 식별자는 특수문자를 제외한 문자, 숫자, `_`, `$` 기호를 사용할 수 있다
2. 단, 숫자로 시작할 수 없다
3. 예약어는 식별자로 사용할 수 없다

- 영어 외의 문자는 권장하지 않는다
- JavaScript는 대소문자를 구별한다
- 변수 이름은 의미가 명확해야 한다 (가독성 향상)

### 네이밍 컨벤션

```javascript
// 카멜 케이스 (camelCase) - 변수, 함수에 사용
var firstName;

// 스네이크 케이스 (snake_case)
var first_name;

// 파스칼 케이스 (PascalCase) - 생성자 함수, 클래스에 사용
var FirstName;

// 헝가리언 케이스 (Hungarian Case)
var strFirstName;
```

- JavaScript에서는 **카멜 케이스**와 **파스칼 케이스**를 주로 사용한다
