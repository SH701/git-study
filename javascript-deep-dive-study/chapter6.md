# 6장 데이터 타입

ES6는 7개의 데이터 타입을 제공한다.

| 구분 | 타입 | 설명 |
|------|------|------|
| 원시 타입 | number | 정수와 실수 구분 없이 하나의 숫자 타입만 존재 |
| | string | 문자열 |
| | boolean | 논리적 참(true)과 거짓(false) |
| | undefined | var 키워드로 선언된 변수에 암묵적으로 할당되는 값 |
| | null | 값이 없다는 것을 의도적으로 명시할 때 사용 |
| | symbol | ES6에서 추가된 7번째 타입 |
| 객체 타입 | object | 객체, 함수, 배열 등 |

> `1`과 `'1'`은 전혀 다르다 (숫자 vs 문자열)

## 6.1 숫자 타입

- JS의 숫자 타입은 **배정밀도 64비트 부동소수점 형식**을 따름
- 모든 수를 실수로 처리 (정수 전용 타입 없음: `int`, `long`, `float` 등 X)
- 2진수, 8진수, 16진수 리터럴도 모두 **10진수로 해석**됨

```javascript
var binary = 0b01000001;  // 2진수 → 65
var octal = 0o101;        // 8진수 → 65
var hex = 0x41;           // 16진수 → 65

console.log(binary === octal);  // true
console.log(octal === hex);     // true
```

### 특수한 숫자 값

| 값 | 설명 | 예시 |
|----|------|------|
| `Infinity` | 양의 무한대 | `10 / 0` |
| `-Infinity` | 음의 무한대 | `10 / -0` |
| `NaN` | 산술 연산 불가 (Not-a-Number) | `1 * 'string'` |

> **주의**: JS는 대소문자를 구분하므로 `NAN`, `Nan`, `nan`은 식별자로 해석됨

## 6.2 문자열 타입

- 0개 이상의 16비트 유니코드 문자(UTF-16)의 집합
- 작은따옴표(`''`), 큰따옴표(`""`), 백틱(``` `` ```)으로 감싸서 표현
- 문자열은 **변경 불가능한 값(immutable)**

```javascript
var str1 = 'Hello';
var str2 = "World";
var str3 = `ES6`;
```

## 6.3 템플릿 리터럴

- ES6에서 도입된 새로운 문자열 표기법
- **백틱(``` `` ```)**을 사용하여 표현

```javascript
var template = `Template literal`;
```

### 6.3.1 멀티라인 문자열

일반 문자열에서는 줄바꿈이 허용되지 않음 → 이스케이프 시퀀스 사용 필요

| 이스케이프 시퀀스 | 의미 |
|------------------|------|
| `\n` | 줄바꿈 (LF) |
| `\t` | 탭 |
| `\\` | 백슬래시 |
| `\'` | 작은따옴표 |
| `\"` | 큰따옴표 |

```javascript
// 일반 문자열
var str = 'Hello\nWorld';

// 템플릿 리터럴 (줄바꿈 직접 가능)
var template = `Hello
World`;
```

### 6.3.2 표현식 삽입

```javascript
var first = 'Ung-mo';
var last = 'Lee';

// 문자열 연결 (+ 연산자)
console.log('My name is ' + first + ' ' + last + '.');

// 템플릿 리터럴 (표현식 삽입)
console.log(`My name is ${first} ${last}.`);
```

## 6.4 불리언 타입

- `true` 또는 `false` 두 가지 값만 존재
- 조건문에서 주로 사용

```javascript
var foo = true;
var bar = false;
```

## 6.5 undefined 타입

- `undefined` 타입의 값은 `undefined`가 유일
- 변수 선언 후 **JS 엔진이 암묵적으로 초기화**할 때 사용하는 값

```javascript
var foo;
console.log(foo);  // undefined
```

> **선언 vs 정의**: ECMAScript에서 변수는 "선언", 함수는 "정의"라고 표현

## 6.6 null 타입

- `null` 타입의 값은 `null`이 유일
- **값이 없음을 의도적으로 명시**할 때 사용
- 함수가 유효한 값을 반환할 수 없는 경우에도 `null` 반환

```javascript
var foo = 'Lee';
foo = null;  // 이전 참조를 제거, 더 이상 'Lee'를 참조하지 않음
```

> **주의**: `null`, `Null`, `NULL`은 모두 다름 (대소문자 구분)

## 6.7 심벌 타입

- ES6에서 추가된 **변경 불가능한 원시 타입**
- 다른 값과 절대 중복되지 않는 유일무이한 값
- 주로 객체의 **프로퍼티 키**로 사용

```javascript
var key = Symbol('key');
var obj = {};

obj[key] = 'value';
console.log(obj[key]);  // value
```

## 6.8 객체 타입

- JS는 객체 기반 언어이며, **JS를 이루고 있는 거의 모든 것이 객체**
- 원시 타입을 제외한 나머지 값(함수, 배열, 정규표현식 등)은 모두 객체 타입

## 6.9 데이터 타입의 필요성

### 6.9.1 데이터 타입에 의한 메모리 공간의 확보와 참조

- 값을 저장할 때 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정됨
- 숫자 타입: 8바이트 메모리 공간 확보

### 6.9.2 데이터 타입에 의한 값의 해석

- 동일한 2진수라도 데이터 타입에 따라 해석이 달라짐

```
0100 0001 → 숫자로 해석: 65
0100 0001 → 문자로 해석: 'A'
```

### 데이터 타입이 필요한 이유

1. 값을 저장할 때 확보해야 하는 **메모리 공간의 크기**를 결정하기 위해
2. 값을 참조할 때 한 번에 읽어 들여야 할 **메모리 공간의 크기**를 결정하기 위해
3. 메모리에서 읽어 들인 **2진수를 어떻게 해석**할지 결정하기 위해

## 6.10 동적 타이핑

### 6.10.1 동적 타입 언어와 정적 타입 언어

| 구분 | 정적 타입 언어 | 동적 타입 언어 |
|------|---------------|---------------|
| 타입 결정 시점 | 변수 선언 시점 | 값 할당 시점 |
| 타입 변경 | 불가능 | 자유롭게 가능 |
| 대표 언어 | C, C++, Java, Kotlin, Go | JavaScript, Python, Ruby |

```javascript
var foo;
console.log(typeof foo);  // undefined

foo = 3;
console.log(typeof foo);  // number

foo = 'Hello';
console.log(typeof foo);  // string

foo = true;
console.log(typeof foo);  // boolean
```

> **동적 타이핑**: 선언이 아닌 **할당에 의해 타입이 결정**되고, 재할당에 의해 언제든지 변할 수 있는 특징

### 6.10.2 동적 타입 언어와 변수

**장점**: 유연성이 높음 (어떤 타입의 값이라도 자유롭게 할당 가능)

**단점**: 신뢰성이 떨어짐 (타입이 언제든 변경될 수 있어 추적이 어려움)

### 변수 사용 시 주의사항

1. 변수는 꼭 필요한 경우에 한해 제한적으로 사용
2. 변수의 스코프를 최대한 좁게 만들어 부작용 억제
3. 전역 변수는 최대한 사용하지 않기
4. 변수보다는 상수(`const`) 사용 권장
5. 변수 이름은 목적과 의미를 파악할 수 있도록 네이밍

> **가독성이 좋은 코드가 좋은 코드다.**
