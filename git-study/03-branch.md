# Git Branch (브랜치) 정리

## Branch란?

**독립적인 작업 공간을 만드는 것**

- 원본 코드를 건드리지 않고 새로운 기능을 개발할 수 있는 평행 세계
- 하나의 프로젝트에서 여러 작업을 동시에 진행 가능

## 왜 사용하냐?

- 새로운 기능 개발 중에도 원본 코드는 안전하게 유지
- 여러 사람이 동시에 다른 기능 개발 가능
- 실험적인 코드도 안전하게 테스트 가능

## 기본 개녕

### Main Branch

- 프로젝트의 **메인 브랜치**
- 배포 가능한 안정적인 코드가 있는 곳
- 직접 수정하지 않고 다른 브랜치에서 작업 후 병합

### Feature Branch

- 새로운 기능을 개발하기 위한 브랜치
- 작업 완료 후 main에 병합

## Branch 네이밍 규칙

```
feat/기능명     # 새 기능 개발
fix/버그명        # 버그 수정
hotfix/긴급수정   # 긴급 패치
release/버전      # 릴리즈 준비
```

예시:

- `feat/user-login`
- `fix/navbar-bug`
- `hotfix/security-patch`

## 핵심 포인트

- 각 브랜치는 **독립적인 commit 히스토리**를 가짐
- 브랜치 간 이동해도 파일은 그대로, **보이는 내용만** 바뀜
- 작업 완료 후 merge로 합치면 하나의 히스토리로 통합됨
