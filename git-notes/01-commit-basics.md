# Git Commit 정리

## Commit이란?

## 핵심개념

- 전체 프로젝트의 상태를 그 시점에 찍어둔 것
- 나중에 그 시점으로 돌아갈 수 있는 세이브 포인트

## 비유

- 게임 세이브 파일처럼 여러 시점을 저장해두고 원하는 시점으로 돌아갈 수 있음

## 중요한 점

-Commit 메세지는 무엇을 변경했는가 보다 **이 시점에 프로젝트가 어떤 상태인가**를 기록

## 이것으로 할 수 있는 것

- `git log`: 히스토리 확인
- `git checkout`: 특정 시점으로 이동
- `git revert`: 특정 시점을 되돌림
